## CRITICAL INSTRUCTIONS FOR FUTURE AGENTS

### About feature_list.json

**⚠️ ABSOLUTELY CRITICAL ⚠️**

1. **NEVER REMOVE TESTS**: Tests in feature_list.json must NEVER be removed
2. **NEVER EDIT TEST DESCRIPTIONS**: Test descriptions are permanent
3. **NEVER MODIFY TEST STEPS**: Testing steps must not be changed
4. **ONLY UPDATE "passes" FIELD**: Change from false to true when test passes

### Why This Is Critical

The feature_list.json file is the **single source of truth** for:
- What needs to be built
- How to test it
- Whether it's complete
- Progress tracking

Removing or editing tests would:
- ❌ Cause features to be forgotten
- ❌ Break progress tracking
- ❌ Lead to incomplete implementation
- ❌ Violate the contract with the user

### How to Use feature_list.json

**Workflow for each test:**
1. Read the test description
2. Read the testing steps
3. Implement the feature
4. Follow the testing steps exactly
5. If all steps pass, mark "passes": true
6. Commit the change
7. Move to next test

**Priority:**
- Work through tests in order (they're prioritized)
- Complete foundational features before advanced ones
- Test thoroughly before marking as passing
- Never skip steps in the testing process
